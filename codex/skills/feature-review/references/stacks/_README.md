# 스택 프로파일 가이드

## 개요

스택 프로파일은 특정 기술 스택에서 이 스킬이 참조하는 구체적 정보를 담는다.
SKILL.md는 기술 스택에 무관한 **프로세스**(분석 → 설계 → 리뷰 → 플랜)를 정의하고, 스택별 상세(외부 스킬, 아키텍처 패턴, 규칙 파일 경로)는 이 디렉토리의 프로파일에 분리한다.

프로파일이 없는 스택에서도 스킬은 정상 동작한다 — CLAUDE.md의 기술 스택 정보를 fallback으로 사용한다.

## 스택 감지 흐름

1. 프로젝트 CLAUDE.md에서 기술 스택 정보(보통 테이블 형태)를 읽는다
2. 이 디렉토리에서 매칭되는 프로파일을 찾아 Read한다
3. 매칭 프로파일이 없으면 CLAUDE.md의 정보만으로 진행한다

## 매칭 규칙

| CLAUDE.md 키워드 | 프로파일 |
|-----------------|---------|
| Next.js + Supabase | `next-supabase.md` |
| Next.js + Firebase | `next-firebase.md` |

## 새 프로파일 추가

1. `<stack-name>.md` 파일을 이 디렉토리에 생성한다
2. 아래 섹션을 포함한다:
   - **외부 스킬**: 기획 시 적용할 외부 스킬 목록
   - **아키텍처 패턴**: 레이어 구조, 의존성 방향, 파일 배치 규칙
   - **규칙 파일 참조**: `.claude/rules/` 내 확인할 파일과 각 파일의 역할
   - **코드 컨벤션**: import 방식, 네이밍, 파일 크기 제한 등
   - **서버 로직 패턴**: API 라우팅, 데이터 액세스 방식
3. 위 매칭 규칙 테이블에 항목을 추가한다
