# UI 사용성 기준 (Service Quality Bar)

> **에이전트가 모바일/웹 UI를 만들거나 수정할 때 반드시 만족해야 할 사용성 기준.**
> 이 기준을 충족하지 못한 화면은 "동작은 하지만 안 좋은 느낌"이 된다. 빡차 추천 페이지 시행착오로 도출된 최소 기준.

## 1. Dead affordance 금지

- "준비 중"이라며 비활성 상태로 자리 차지하는 버튼/네비/아이콘 금지
- 누를 수 있게 보이는데 안 눌리는 컨트롤은 사용자 신뢰를 깎는다
- 동작하지 않는 컨트롤은 **아예 빼라**. 굳이 자리만 유지하려는 "준비중" 라벨도 노이즈

## 2. 정보 밀도와 카드 사이즈

- 모바일 한 화면(약 600px 가용)에 핵심 항목 **3개 이상**이 들어와야 한다
- 풀블리드 hero 카드는 페이지 메인 **1장**만. 본격 리스트는 컴팩트 가로형(썸네일 좌측 ~100px + 정보 우측, 카드 ~140-180px)
- "한 카드로 다 표현"하려고 hero + 인용 + CTA를 모두 넣지 말 것. 상세 정보는 **상세 페이지**에서

## 3. 가짜 통계/숫자 금지

- 폴백 mock 숫자(예: "검수 247건", "가이드 12개")는 표시하지 않는다
- 실데이터가 없으면 그 통계 자체를 표시 안 함
- 사용자 신뢰는 fake 한 줄에도 깨진다

## 4. 가로 스크롤 풀블리드

[[feedback-horizontal-scroll-fullbleed]] 참조. 부모 padding 영향을 받지 않게 풀블리드(`marginHorizontal: -X`) + content padding(`paddingHorizontal: X`).

## 5. Active scroll-into-view

가로 chip/탭에서 현재 active 항목은 자동으로 가운데로 scroll. 양 끝 항목은 패딩 영역까지만 흐름. `onLayout` + `ScrollView.scrollTo` 패턴.

## 6. 인라인 expand (팝오버 X)

선택/필터 UI는 모달/팝오버 대신 **같은 자리에서 펼쳐지는** 인라인 expand가 사용자 컨텍스트를 보존한다. 빡차 주차 선택 picker 패턴.

## 7. 외부 인풋 → 모든 관련 UI 자동 동기화

chevron 같은 외부 컨트롤로 상태가 바뀌면, 펼친 패널의 active, scroll 위치, 필터 결과 등 모든 관련 UI가 자동으로 따라온다. 사용자가 같은 정보를 두 번 입력하게 만들지 말 것.

## 8. 노이즈 제거

- 매거진 톤은 좋지만, 한 페이지에 여러 톤(hero + journal + lookbook + 다시 hero CTA)이 섞이면 정보 위계가 무너진다
- 한 페이지는 하나의 미션. 그 미션에 기여하지 않는 섹션은 빼라
- 푸터 CTA는 페이지당 **1개 이하**. 카드 안에 이미 진입 액션이 있는데 푸터 CTA를 또 두는 건 중복

## 9. 한 페이지 한 페이지 헤더

- 탭 루트가 아닌 sub 화면은 SubAppBar (제목 + 뒤로가기)만
- 우상단 검색/도움말/알림 등 추가 액션은 **기본 금지** — 실제 동작하는 메인 기능에만 허용
- 헤더는 가벼울수록 좋다

## 10. RN + NativeWind 환경 layout 안정성

[[feedback-rn-web-flex]] 참조. `flexDirection` 등 layout-critical 속성은 className 대신 inline style. 함수형 style에 박은 flexDirection이 RN-Web에서 안 먹는 케이스 회피.

## How to apply

- 화면 작성/수정 후 위 10개 기준을 self-check
- 새 UI 패턴(가로 스크롤, 선택, expand 등) 작성 시 같은 패턴이 다른 화면에 이미 있으면 **그걸 먼저 참조**
- 죽은 affordance를 발견하면 즉시 제거 또는 의미 있게 재배치
- 디자인 톤(매거진/모던/심플)과 사용성 기준은 **둘 다** 충족해야 한다. 디자인 우선이라며 사용성을 희생하면 페이지 가치가 떨어진다
- 단순한 페이지가 좋은 페이지다. "정보가 많아서" 카드가 큰 것과 "여러 톤을 다 보여주려서" 페이지가 긴 것은 다르다 — 후자만 정리한다
